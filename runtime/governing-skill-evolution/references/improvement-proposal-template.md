# Improvement Proposal Template

```yaml
artifact_type: improvement-proposal
artifact_version: "1.0"
case_id:
source_skill: skill-analysis-suite
generated_at:
status: proposed
owner:
```

Use `unknown` for unavailable values; all fields are required.

## 1. General problem and failure symptoms
## 2. Case evidence and occurrence scope
## 3. Proposed change and affected Skills/references
## 4. Context cost, compatibility, and false-trigger risk
## 5. Old-fail/new-pass regression
## 6. Target: central candidate or explicitly selected personal branch
## 7. Maintainer, approval, and release decision
## 8. Rollback target

If evidence comes from one ordinary case, state that central-entry criteria are not met.
