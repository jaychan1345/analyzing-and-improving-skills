# Release Checklist

- [ ] Five runtime Skills pass structural validation.
- [ ] Positive and adjacent negative trigger cases are distinct.
- [ ] Missing references remain missing or unverified, never inferred as fact.
- [ ] A change plan and explicit approval precede derivation.
- [ ] A derived Skill has a new name and source provenance.
- [ ] Failed or unexecuted tests block release claims.
- [ ] A normal single case creates only a proposal for the central suite.
- [ ] Personal branches require explicit selection and distinct names.
- [ ] Runtime installation excludes governance files.
- [ ] The share tree contains no business data, secrets, accounts, or machine-specific paths.
